# 🔐 Phase 2: Cloud Platform Security (AWS & Azure)

Building upon the fundamental cloud concepts from Security+, this module gets hands-on with the two major cloud service providers. You will learn how to design, implement, and maintain a secure cloud architecture.

### 🎯 Objectives
By the end of this module, you will understand:
* Cloud Identity and Access Management (IAM) at scale.
* Securing virtual networks (VPCs and Vnets).
* Data encryption at rest and in transit using cloud-native key management.
* Cloud Security Posture Management (CSPM) and continuous monitoring.

---

### 📚 Study Checklist

#### 1. Identity and Access Management (IAM)
* [ ] Read: Principle of Least Privilege in cloud environments
* [ ] Lab: Configuring AWS IAM Roles, Policies, and Azure Role-Based Access Control (RBAC)
* [ ] Lab: Implementing Multi-Factor Authentication (MFA) for root and administrative accounts

#### 2. Network and Infrastructure Security
* [ ] Read: Designing secure Virtual Private Clouds (VPCs) and Azure Virtual Networks (VNets)
* [ ] Lab: Configuring Network Security Groups (NSGs) and Application Security Groups (ASGs)
* [ ] Lab: Setting up a Web Application Firewall (WAF) to protect cloud-hosted web apps

#### 3. Data Protection and Encryption
* [ ] Read: Shared Responsibility Model deep dive
* [ ] Lab: Managing encryption keys using AWS KMS or Azure Key Vault
* [ ] Lab: Enforcing server-side encryption for cloud storage (Amazon S3 / Azure Blob Storage)

#### 4. Logging, Monitoring, and Incident Response
* [ ] Read: Cloud-native threat detection (Amazon GuardDuty / Microsoft Defender for Cloud)
* [ ] Lab: Setting up centralized logging using AWS CloudTrail and Azure Monitor
* [ ] Lab: Automating incident response with serverless functions (AWS Lambda / Azure Functions)
